## 🗄️ Supabase配置

### 1. 创建项目
1. 访问 https://supabase.com/dashboard
2. 点击 "New Project"
3. 填写项目信息：
   - **Name**: vehicle-query
   - **Database Password**: 设置强密码（请保存）
   - **Region**: 选择距离你最近的区域

### 2. 创建数据表
在Supabase的 SQL Editor 中执行以下SQL：

```sql
-- 创建车辆信息表
CREATE TABLE vehicle_infos (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  owner_name VARCHAR(100) NOT NULL,
  license_plate VARCHAR(20) NOT NULL UNIQUE,
  phone VARCHAR(20),
  department VARCHAR(50),
  description TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引
CREATE INDEX idx_vehicle_infos_license_plate ON vehicle_infos(license_plate);
CREATE INDEX idx_vehicle_infos_department ON vehicle_infos(department);
CREATE INDEX idx_vehicle_infos_created_at ON vehicle_infos(created_at DESC);

-- 插入示例数据（可选）
INSERT INTO vehicle_infos (owner_name, license_plate, phone, department, description) VALUES
('张三', '京A12345', '13800138000', '教师车辆', '黑色轿车'),
('李四', '京A54321', '13900139000', '后勤车辆', '白色面包车');
```

### 3. 获取Supabase配置信息
1. 进入项目设置 → API
2. 记录以下信息：
   - **Project URL**: `https://xxx.supabase.co`
   - **anon public key**: `eyJxxx...`

## 🚀 Vercel部署

### 1. 连接Git仓库

#### 方式A：通过Vercel网站（推荐）
1. 访问 https://vercel.com/dashboard
2. 点击 "Add New Project"
3. 导入你的Git仓库（GitHub/GitLab/Bitbucket）
4. Vercel会自动检测项目配置

#### 方式B：通过CLI
```bash
cd /workspace/projects
vercel
```
按照提示操作，选择项目配置。

### 2. 配置环境变量

在Vercel项目设置中添加以下环境变量：

**项目设置 → Environment Variables**

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `SUPABASE_URL` | `https://xxx.supabase.co` | Supabase项目URL |
| `SUPABASE_ANON_KEY` | `eyJxxx...` | Supabase匿名密钥 |
| `COZE_BUCKET_ENDPOINT_URL` | `https://xxx` | 对象存储端点（如需要） |
| `COZE_BUCKET_NAME` | `your-bucket` | 对象存储桶名（如需要） |
| `EXPO_PUBLIC_BACKEND_BASE_URL` | `/api` | API路径（固定值） |

### 3. 配置构建命令

在Vercel项目设置中配置：

**Root Directory**: 留空（根目录）

**Build Command**:
```bash
pnpm install --filter client && pnpm install --filter server
```

**Output Directory**: `client/dist`（Web构建输出）

**Install Command**:
```bash
pnpm install
```

### 4. 部署

#### 方式A：自动部署
- Git push到仓库后，Vercel会自动部署

#### 方式B：手动部署
```bash
# 使用部署脚本
./deploy.sh

# 或直接使用Vercel CLI
vercel --prod
```

## 🔧 常见问题

### 问题1：部署失败 - 依赖安装错误
**解决方案**：
```bash
# 确保根目录有package.json
# pnpm workspaces配置正确
```

### 问题2：API返回404
**解决方案**：
- 检查 `vercel.json` 配置是否正确
- 确保API文件路径正确：`server/api/v1/vehicles.ts`

### 问题3：数据库连接失败
**解决方案**：
- 检查Supabase环境变量是否正确配置
- 确认Supabase项目是否已激活
- 检查Supabase RLS（行级安全）策略

### 问题4：文件上传失败
**解决方案**：
- 检查Vercel函数的超时设置（默认10秒）
- 可能需要升级Vercel计划以获得更长的执行时间
